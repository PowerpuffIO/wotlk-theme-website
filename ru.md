# Acore-www — сайт для приватных серверов WoW 3.3.5a

Лёгкий PHP-фронтенд для реалмов на **AzerothCore** (или совместимых): регистрация по **SRP6**, новости, загрузка клиента, статус реалма, ладдер игроков, голосование (MMORating), тикеты и админ-панель. Интерфейс **RU/EN** (переключатель языка в шапке).

## Возможности и страницы

| Раздел | Описание |
|--------|----------|
| **Главная** | Лента новостей, статус реалма и доля фракций, боковая колонка **Social** (зарегистрированные пользователи, персонажи онлайн, виджет Discord). |
| **Новости** | `/news/{id}` — полная статья с картинкой. Вошедшие пользователи могут ставить лайк/дизлайк. |
| **Скачать** | Ссылки на торрент/прямую загрузку, копирование realmlist — строки задаются в админке. |
| **Статистика (ладдер)** (`/ladder`) | Топ по времени в игре, почётным победам, арены 2×2 / 3×3 / 5×5 (таблицы `characters` / `arena_team`). |
| **Регистрация / вход** | Аккаунт на сайте связан с игровым (SRP6 + расширение `gmp`). Капча (Google reCAPTCHA или Cloudflare Turnstile) настраивается. |
| **Сброс пароля** | Ссылка в письме (SMTP по желанию). |
| **Профиль** | Обзор, число персонажей из игровой БД. |
| **Настройки** | Смена email, запрос ссылки на смену пароля. |
| **Голосование** | Проверка голоса через API MMORating.top + бонусный баланс (в админке: API-ключ, размер бонуса). |
| **Сообщения** | Тикеты пользователей; ответы персонала в админке. |
| **Админ-панель** | Сводка, новости CRUD (с загрузкой картинок), настройки голосования/MMORating, строки для страницы загрузки и realmlist, тикеты, **Social** (ID сервера Discord + тема виджета). |

## Требования

- **PHP** 7.4+ с расширениями: `pdo_mysql`, `json`, `session`, `mbstring`, **`gmp`** (обязательно для регистрации SRP6), при необходимости `openssl`.
- **MySQL** / MariaDB — три логические базы: **сайт** (это приложение), **auth**, **characters** (AzerothCore).
- Веб-сервер с **переписыванием URL** на `index.php` (см. `.htaccess` для Apache; для nginx — аналог `try_files`).

## Установка

1. **Загрузите** проект в корень сайта (или подкаталог; укажите `BASE_PATH` в `configs/config.php`, если сайт не в корне домена).

2. **Создайте** базу MySQL для сайта и импортируйте схему:
   ```bash
   mysql -u USER -p SITE_DB_NAME < sql/install.sql
   ```

3. **Отредактируйте** `configs/config.php`:
   - `SITE_DB_*` — база сайта (из шага 2).
   - `AUTH_DB_*` — база **auth** AzerothCore.
   - `CHARACTERS_DB_*` — база **characters** AzerothCore.
   - `SERVER_NAME`, `REALM_NAME`, `REALM_RATE`, `UPTIME_REALM_ID`, `SITE_PUBLIC_URL` (публичный URL для писем и ссылок).
   - Почта: `MAIL_*` и при необходимости SMTP.
   - По желанию: `CAPTCHA_*` (см. комментарии в конфиге), ключи MMORating задаются в **Админка → Голосование**.

4. **Права** — пользователь веб-сервера должен иметь запись в `storage/news/` (картинки новостей).

5. **Веб-сервер** — корень документов на эту папку; включите **mod_rewrite** (Apache) или настройте nginx `try_files` на `index.php`.

6. **Администратор** — выставьте `role = 1` у нужного пользователя в таблице `users` (БД сайта), чтобы открывался `/profile/adminpanel`.

7. Если база уже была со старой схемой, при необходимости выполните `sql/migration_discord_widget.sql` для ключей настроек Discord.

Откройте сайт, зарегистрируйте тестовый аккаунт и войдите в админку, чтобы добавить новости, ссылки на загрузку, ID сервера Discord (Настройки сервера → Виджет в Discord) и API-ключ MMORating.
